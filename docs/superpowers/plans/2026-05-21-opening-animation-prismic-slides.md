# Opening-Animation Prismic Slides Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> **Repo convention:** This repo's owner prefers to run `git commit` themselves. Each task includes a commit step; pause and show the diff for approval before running `git commit`.

**Goal:** Move the 4 hardcoded rotating slides in `OpeningAnimation.svelte` into a new Prismic single-type custom type so content editors can manage them without a deploy.

**Architecture:** Add a new `opening_animation` Prismic single-type (parallel to `logo_soup`) with a `slides` group field. Load it server-side on the home route and pass to the component as a typed prop. Replace the build-time-imported `<Img>` calls with `<PrismicImage>`, matching the pattern already used in `LogoSoup.svelte`.

**Tech Stack:** SvelteKit + Svelte 5 (runes), Prismic (`@prismicio/client` + `@prismicio/svelte`), Slice Machine, TypeScript, pnpm, Tailwind v4.

**Spec:** `docs/superpowers/specs/2026-05-21-opening-animation-prismic-slides-design.md`

**Note on TDD:** This project has no component-level unit tests (only Playwright smoke tests). "Verification" steps use `pnpm check` for types, `pnpm build` for compile, and manual browser inspection at `http://localhost:5173/` for visual confirmation. There is no failing-test-first cycle for these tasks.

    fetchpriority={index === 0 ? "high" : "auto"}
